---

## Что делает система

Это интерактивная web-инсталляция — цикл из четырёх фреймов, через который зритель проходит в браузере. На экране — вхождение с портретом, миниатюра «Сотворение жизни», портрет «Неприкосновенна» и финальное «кровавое» состояние. Зритель кликает, ведёт мышью, нажимает кнопки — и каждая страница реагирует. Клики по портрету складываются в общую видимую карту прикосновений.

---

## Из чего состоит

- **Четыре фрейма** — вхождение с портретом Джоконды, миниатюра «Сотворение жизни» с окном глаз, руками, ч/б фотографией и видео, портрет «Неприкосновенна», финал с видео-трансформацией и «кровью».
- **Кастомный курсор** — у зрителя свой курсор с физикой движения, который меняет иконку в зависимости от того, на что он наведён (фон / портрет / кнопка).
- **Карта прикосновений** — клики зрителей по портрету сохраняются и видны как след. Новые точки появляются сразу.

---

## Почему так сделано

- **Никаких подсказок и инструкций.** Зритель сам разбирается. Это художественное решение — инсталляция о прикосновении, а не о юзабилити.
- **Минимум UI.** На экране только то, что нужно для переживания: портрет, текст, две-три кнопки. Никаких меню, шапок, кнопок «поделиться».
- **Со-присутствие через карту.** Зритель не один. Видит чужие следы — это переводит работу из созерцания в участие.
- **Технически — два простых контейнера.** Один отдаёт страницу, второй хранит точки и раздаёт курсоры. Без облаков, без сторонних сервисов, без аккаунтов.

---

## Стек

- Веб-приложение на React (фронт) и Node.js + Express (бэк).
- База данных — SQLite (один файл на сервере, без отдельной СУБД).
- ИИ-моделей нет.
- Развёрнуто на одном сервере, доступ через браузер по ссылке.
- Текущая техническая версия сцен — `v0`; публичные ссылки для зрителя не содержат этого префикса.

---

## Доменные термины

- **Триптих / цикл фреймов** — условное название последовательных частей одной работы; в текущей версии их четыре.
- **Миниатюра** — маленькая самостоятельная сцена внутри триптиха.
- **Отпечаток / прикосновение** — клик зрителя по портрету, который сохраняется в общей карте.
- **Сакральное** и **рукотворное** — ключевая оппозиция работы: то, к чему нельзя прикасаться, и то, что сделано руками. Работа о том, что это одно и то же.

## Что система может и чего НЕ может

### Может

- Открываться по прямой ссылке на десктопе и работать без установки.
- Проводить зрителя через четыре сцены триптиха с переходами по клику.
- Сохранять все клики всех зрителей в общей карте и показывать их новым посетителям.
- Запоминать «кровавое» состояние финала для конкретного зрителя между визитами.

### НЕ может

- Не работает корректно с телефона / планшета — нет touch-жестов, расчёт на мышь и большой экран.
- Нет аккаунтов, регистрации, личных кабинетов — это не задумано, это арт-проект.
- Нет монетизации, нет рекламы, нет аналитики поведения.
- Нет мультиязычности — только русский.
- Публичные курсоры других зрителей в активной версии `v0` не показываются, хотя технический слой для них есть в коде.
- Нет резервных копий БД с отпечатками — если файл потеряется, карта обнулится. Это осознанный риск стадии.

## Как хранятся данные

В системе одна простая сущность — отпечаток. Это просто координаты точки на портрете.

### Отпечаток

```json
{
  "x": 52.4,
  "y": 38.1
}
```

Координаты — это проценты от ширины и высоты области портрета. Все отпечатки хранятся в одной таблице, без привязки к зрителю и без времени.

## Демо-сценарии

### Сценарий 1: пройти триптих

**Действие:** зритель открывает `/`, видит портрет Джоконды и две кнопки — «плюнуть» и «поцеловать». Нажимает обе, затем на `/01_01` кликает окно с глазами и кнопку «неприкосновенна» в зоне, не перекрытой руками.

**Ожидаемый результат:** через одну секунду после второго клика зрителя перебрасывает на миниатюру «Сотворение жизни». После успешного клика на кнопку курсор и параллакс замирают, через 0.5 секунды старые слои исчезают, появляется ч/б фотография, затем за 2 секунды она растворяется и открывает играющее видео.

**Что доказывает:** первые два фрейма и сцены 05–06 работают без подсказок.

### Сценарий 2: оставить след

**Действие:** на портрете «Неприкосновенна» зритель кликает несколько раз в разных местах.

**Ожидаемый результат:** каждая точка появляется на портрете как след прикосновения. После перезагрузки страницы все эти точки остаются — и к ним добавлены точки других зрителей за всё время.

**Что доказывает:** карта прикосновений работает, отпечатки персистентны.

### Сценарий 3: увидеть финальное состояние

**Действие:** зритель открывает финальный URL и нажимает кнопку «неприкосновенна».

**Ожидаемый результат:** запускается видео-трансформация, фон переходит в «кровавое» состояние, а состояние сохраняется для этого браузера между визитами.

**Что доказывает:** финальный фрейм и локальное сохранение состояния работают.
